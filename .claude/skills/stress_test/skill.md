# ストレステストスキル

8つのシナリオで銘柄のリスクを検証するスキル。

## 使い方

ユーザーが「リスクを検証して」「ストレステストして」「暴落シミュレーション」等と指示した場合にこのスキルを使用。

## 実行方法

```bash
cd /home/user/screening-test
uv run screening-test stress {ticker}
```

## パラメータ

- `ticker`: ストレステスト対象のティッカーシンボル

## 8つのシナリオ

1. 金利上昇（中央銀行が政策金利を1%引き上げ）
2. 景気後退（GDPが2四半期連続マイナス）
3. 為替急変・円高（ドル円が20円下落）
4. パンデミック（新たなパンデミック発生）
5. 地政学リスク（主要地域での紛争拡大）
6. セクターショック（業種特有の規制強化）
7. 流動性危機（信用市場の急激な収縮）
8. テクノロジーバブル崩壊（ハイテク株の大幅調整）

## セクター感応度

セクターに応じてショックの影響度が自動調整される。例えば:
- テクノロジー: 感応度1.3（ショック拡大）
- 生活必需品: 感応度0.6（ショック縮小）
- ヘルスケア: 感応度0.8（比較的安定）

## 使用例

- 「トヨタのリスクを検証して」→ `stress 7203.T`
- 「NVIDIAのストレステスト」→ `stress NVDA`
